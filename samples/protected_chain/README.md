# Protected Sample Chain

This sample is an authorized-software protection exercise. It builds a local VM-protected artifact from the C++ core/runtime skeletons, executes the protected behavior against a baseline, and emits machine-readable QA reports.

Run from the repository root:

```bash
tests/integration/run_protected_sample_chain.sh
```

Generated files are written under `samples/protected_chain/out/`:

- `protected_sample.vmp`: release-style sample artifact containing VM metadata and encrypted bytecode only.
- `behavior.json`: baseline/protected consistency cases.
- `strings.json`: critical string absence checks for the release artifact.
- `randomness.json`: deterministic rebuild and seed-divergence checks.
- `docs/qa/reports/performance-sample.json`: baseline/protected timing and artifact-size report generated by `tests/performance/protected_sample_benchmark.sh`.
- `artifacts/protected/linux/protected_release_sample`: stripped local Linux executable embedding and executing the encrypted VM sample, generated by `tests/integration/run_release_protected_binary.sh`.

The protected artifact intentionally does not store critical marker strings or the build seed.
