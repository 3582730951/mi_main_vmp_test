# 任务级计划：对标 VMProtect 梯队的跨平台 LLVM IR 多层 VMP

## Summary

建设一个以 LLVM IR 为核心的多层 VMP：OLLVM IR 混淆、多虚拟机嵌套、随机 opcode、花指令、反静态/动态分析、平台适配和 CI 验收。`main_agent` 统一监督，平台 agent 并行开发。验收目标是商用强保护，对标 VMProtect 的 virtualization、mutation、combined protection 能力，同时以“防御优先，其次性能，再其次资源占用”为全局优先级。

本计划只适用于保护自有或授权软件。所有反调试、反注入、反分析能力只作为软件防护和完整性验证使用，不实现持久化隐藏、内核 rootkit、恶意规避安全产品或未授权绕过。

## Hard Acceptance

| 项目 | 验收标准 |
|---|---|
| Android | 加密/保护后的 APK 或 `.so` 必须能在 Android 模拟器内正常运行，通过 smoke test、JNI 调用测试和核心逻辑一致性测试。 |
| iOS | 当前阶段只做逻辑可运行验证，要求 Mach-O/iOS runtime 设计、签名流程和 no-JIT 约束在逻辑上可成立；真机执行不作为当前阶段硬门槛。 |
| Windows | 非 Linux 平台编译和保护后运行测试必须走 GitHub Actions CI；Windows protected `.exe/.dll` 必须在 CI 中执行成功。 |
| CI | Windows/macOS 使用 GitHub-hosted 或 self-hosted runners；敏感信息只能使用 GitHub Encrypted Secrets，不允许明文使用 `passwd.txt` 内容。 |
| 对标 | 保护能力必须进入 VMProtect 同梯队，最低包含代码虚拟化、变异混淆、组合保护、字符串隐藏、导入隐藏、反调试、反注入、反篡改。 |
| 字符串 | release protected binary 中不得存在业务关键字符串、API 名称、JNI 名称、授权字段、URL、密钥材料的明文。 |
| IDA/OllyDbg | 目标是让关键函数 F5、函数识别、xref、调用图、字符串引用严重失真；不承诺数学意义“完美”，验收以自动化指标和人工复核通过为准。 |
| 优先级 | 防御强度最高，其次速度/性能，再其次硬件资源占用；若三者冲突，默认牺牲性能和体积保证防御。 |
| 动态对抗 | 必须检测非程序自身设置的硬件断点、内存断点、调试器、DLL/SO 注入、Android Xposed/LSPosed 类框架、root 环境、可疑 ptrace/hook/frida 行为。 |
| 合法边界 | 仅用于保护自有或授权软件；不实现绕过系统安全产品、持久化隐藏、内核 rootkit 或恶意规避能力。 |

## Global Execution Rules

| 规则 | 内容 |
|---|---|
| R1 | `main_agent` 拥有接口、优先级、合并、回滚和任务重排决策权。 |
| R2 | 子 agent 遇到 ABI、bytecode、配置字段、runtime 行为冲突时必须停下上报。 |
| R3 | 每个任务交付必须包含代码/文档改动、测试命令、失败项、外部资料链接。 |
| R4 | 平台 agent 只能改自己平台目录和测试，不允许改 core ABI。 |
| R5 | 联网查阅以官方文档、平台源码、标准规范、release notes 和实测结果为准；博客和论坛只能作为线索。 |
| R6 | `passwd.txt` 只能作为凭据迁移来源，不允许打印、提交、写入 workflow 或进入日志。 |
| R7 | 所有敏感凭据必须迁移到 GitHub Secrets，CI 中只能通过 `${{ secrets.* }}` 引用。 |
| R8 | 防御和性能冲突时，默认优先防御；只有 `main_agent` 可以调整 profile 默认值。 |

## Main Agent Runtime Protocol

1. 每个 batch 开始前，`main_agent` 下发任务包：owner、scope、allowed files、dependencies、tests、acceptance。
2. 子 agent 每次提交必须包含：改动摘要、文件列表、测试命令、失败项、需要决策的问题、引用资料链接。
3. 子 agent 不得私自修改 `BytecodeSpec.md`、`VMRuntimeABI.md`、`ProtectionConfig.md`；需要修改时上报 `main_agent`。
4. `main_agent` 对 blocker 的处理顺序：复现实测 -> 联网查官方资料 -> 判定接口或局部实现问题 -> 更新任务和文档 -> 重新下发。
5. 资料优先级：官方文档、平台源码、标准规范、release notes、实测结果；博客和论坛只能作为线索。

## Measurable Anti-Analysis Gates

| Gate | 指标 |
|---|---|
| G1 字符串 | `strings`、平台对象扫描、APK/IPA 解包扫描中，关键业务字符串、URL、密钥、授权字段为 0。 |
| G2 随机性 | 同一输入不同 seed，opcode map、handler order、bytecode layout、section layout、junk templates 必须明显不同。 |
| G3 IDA | 关键函数自动 F5 失败或伪代码严重失真，真实调用图不可直接从 xref 追踪。 |
| G4 OllyDbg/x64dbg | 受保护 Windows 样本在调试/断点场景中触发检测或降级策略。 |
| G5 注入 | 异常 DLL/SO、hook 框架、可疑模块映射触发检测，正常环境误报受控。 |
| G6 Android hostile env | root、Xposed/LSPosed、Frida/hook 相关检测在测试样本中可触发。 |
